    ```typescript
    export interface WatcherConfig {
      sourceDirs: Map<SyncSourceType, string[]>;  // source type → watched directory paths
    ```

    ```typescript
    }
    ```

    3. Implement `createWatcher(config: WatcherConfig)` returning:
    ```typescript
    {
      start(): Promise<void>;
      stop(): Promise<void>;
      getStatus(): WatcherStatus;
    }
    ```

    4. Internal state: `pendingPaths: Set<string>`, `debounceTimer: ReturnType<typeof setTimeout> | null`, `resyncInterval: ReturnType<typeof setInterval> | null`, `chokidarWatchers: Map<string, chokidar.FSWatcher>`, `status: WatcherStatus`.

    5. `start()` implementation:
       - For each `[sourceType, dirs]` in config.sourceDirs, for each dir: verify it exists (fs.access), create a chokidar watcher with `{ ignoreInitial: true, depth: 0, awaitWriteFinish: { stabilityThreshold: 300 } }`.
       - On `add`/`change`/`unlink`: filter by fileExtensions (only `.jsonl`, `.json`, `.md`), filter out temp suffixes (`~`, `.swp`, `.tmp`), add to `pendingPaths` Set, reset debounce timer (clearTimeout then setTimeout 500ms).
       - On debounce fire: call `syncSource(sourceType)` for each distinct source type in `pendingPaths`, clear pendingPaths.
       - Start periodic resync: `setInterval(() => runPeriodicResync(), config.resyncIntervalMs)`.
       - Set `status.running = true`, watch error handlers for each chokidar watcher.

    6. `runPeriodicResync()`: for each source type, call `syncSource(sourceType)` which triggers full directory scan. After sync completes, update `status.lastSyncAt`.

    7. `stop()`: clear all intervals/timeouts, close all chokidar watchers.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → ingest watcher | chokidar watches arbitrary paths on disk |
| ingest watcher → sync pipeline | file paths passed to parsers |
| ingest API → watcher status | status data exposed via REST |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | watcher debounce | mitigate | pendingPaths Set clears after each sync batch; no file writes from watcher |
| T-06-01-02 | Denial of Service | chokidar watch dirs | mitigate | Extension filter (.jsonl/.json/.md only) prevents temp-file flood; depth: 0 option prevents recursive watching |
| T-06-01-03 | Information Disclosure | sync_status API | mitigate | No file paths exposed in error messages when INGEST_DEBUG is not true |
| T-06-01-04 | Elevation of Privilege | file hash computation | accept | SHA-256 hash only used for dedup, no auth decisions |
</threat_model>
