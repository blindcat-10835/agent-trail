## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser -> BFF | Aggregate hook may only fetch `/api/agent-tools/[tool]/sessions`. |
| Client state -> UI | Pagination state controls how many rows are appended and displayed. |
| BFF -> ingest | Per-source pagination params must remain bounded by server adapter validation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-04 | Elevation of Privilege | `useAggregateSessions` fetch boundary | mitigate | Use `fetchToolApi` only; tests assert BFF URLs. |
| T-09-05 | Denial of Service | aggregate pagination | mitigate | Keep page size per source at BFF-capped limit, fetch only sources with `hasMore`, and avoid raising `MAX_LIMIT`. |
| T-09-06 | Tampering | duplicate session ids across pages | mitigate | Deduplicate by `session.id` before sorting and rendering. |
| T-09-07 | Repudiation | displayed indexed totals | mitigate | Derive totals from source API pagination metadata, not from loaded client rows. |
</threat_model>
