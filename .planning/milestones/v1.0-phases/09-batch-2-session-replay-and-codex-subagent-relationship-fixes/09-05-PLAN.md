<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Local Codex logs -> relationship collection | JSONL events are untrusted local data and may be malformed. |
| Sync backfill -> SQLite | Idempotent UPDATE changes persisted relationship fields for indexed sessions. |
| Session list API -> frontend | Default child filtering depends on relationship columns being correct. |
| Local corpus harness -> repo | User log contents must not be copied into tracked files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-16 | Tampering | `backfillCodexRelationships` | mitigate | Parameterized prepared UPDATE, skip empty/self relationships, update only `source='codex'`. |
| T-09-17 | Repudiation | sync order | mitigate | Tests cover child-before-parent/backfill-after-collection parse order. |
| T-09-18 | Denial of Service | full/background sync | mitigate | Keep limited startup behavior bounded; full sync backfills after existing relationship collection without repeated scans in UI code. |
| T-09-19 | Information Disclosure | local real-session test | mitigate | Opt-in harness reads local files only at test time and commits no log content. |
| T-09-20 | Elevation of Privilege | subagent hiding | mitigate | Use `relationship_type='subagent'` only; do not add `hide_single_turn` heuristic that could hide unrelated sessions. |
</threat_model>
