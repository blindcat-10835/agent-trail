# Plan 08-05: Target-session verification and regression closure

## Goal

Close Phase 8 by proving the user-reported failures are fixed against both deterministic tests and the named local sessions from the investigation.

## Tasks

1. Add non-local regression tests in `tests/unit/ingest/phase8-regression.test.ts`.
   - Use small temporary JSONL files and in-memory DB.
   - Verify parser + sync + assembleTurns end to end for Claude tool_result and Codex function/custom tool outputs.
   - Verify message ids are non-null without relying on frontend key fallback.
2. Extend `tests/local/real-session-corpus.test.ts` target checks.
   - Recognize tags:
     - `claude-key-null-risk`
     - `claude-discoverability`
     - `codex-function-output`
     - `codex-custom-tool`
     - `claude-subagent`
   - For `606dac00-4f36-40e2-89c8-da91416b6b39`: after force sync, assert `count(*) = count(id)` for messages.
   - For `effac644-0eb7-4fc8-9e60-6c8127d51eae`: after force sync, assert session is discoverable via sessions API/query path.
   - For tagged tool sessions: assert `tool_calls` and `tool_result_events` counts are greater than zero.
3. Run force reparse verification.
   - Use BFF/ingest force path from Plan 04 where possible.
   - If running direct ingest tests, call `syncSource(source, { force: true })`.
4. Add `08-VERIFICATION-NOTES.md`.
   - Record exact commands run.
   - Record DB assertion summaries.
   - Record whether local corpus was present or skipped.
   - Do not include raw prompts, tool outputs, or absolute private paths.
5. Manual smoke if dev servers are running:
   - Open target Claude/Codex sessions.
   - Confirm replay tool blocks show inputs/results.
   - Confirm browser console no longer shows `key=null` warning for target session.

## Verification

- `pnpm test:run tests/unit/ingest/phase8-regression.test.ts`
- `pnpm typecheck`
- `pnpm typecheck:ingest`
- `RUN_REAL_SESSION_TESTS=1 pnpm test:real-sessions` if local corpus manifest is present.

## Acceptance Criteria

- The named `606dac00-...` local session has non-null message ids after force reparse.
- The named `effac644-...` Claude session remains discoverable after force reparse.
- Local DB has non-zero `tool_calls` and `tool_result_events` for sessions with real tools.
- Replay receives structured tool activities from `assembleTurns()`.
- Verification notes exist and contain command outcomes without sensitive raw session content.
