# Plan 08-02: Repair Claude and Codex parsers for real JSONL formats

## Goal

Align parser behavior with actual local Claude Code and Codex JSONL records while preserving the canonical trace model.

## Tasks

1. Extend parser activity metadata.
   - Add a parser-owned way to carry `messageOrdinal` on tool calls.
   - Prefer optional fields on `TraceToolCall` only if needed by sync; otherwise add a typed parser helper in `ingest/parser/types.ts`.
   - Include `sourceLine` for diagnostics where useful.
2. Repair Codex parser:
   - Handle `response_item.payload.type === "function_call_output"` and read `payload.output`.
   - Handle `custom_tool_call` as a `TraceToolCall`.
   - Handle `custom_tool_call_output` as result events.
   - Normalize arguments from either `arguments` or `input`.
   - Treat `reasoning` as `TraceThinkingBlock` or a non-noisy system/thinking representation.
   - Treat `web_search_call` as a system/tool activity or silently recognized known type; it must not emit unknown warning spam.
   - Keep real `message` payload support and current synthetic fixture compatibility.
3. Repair Claude parser:
   - Parse content blocks through one shared helper that handles `text`, `thinking`, `tool_use`, and `tool_result`.
   - For assistant `tool_use`, emit `TraceToolCall` with `id = block.id`, `messageOrdinal = assistant message ordinal`, `inputJson`, category, and pending status.
   - For user `tool_result`, decode string/array/object content and attach a `TraceToolResultEvent` to the matching tool call by `tool_use_id`.
   - Tool-result-only user records should not become new user turns. Map them to `tool_result` role or another assembler-safe representation.
   - Handle `isCompactSummary` and file-history snapshot records as compact/system boundaries; do not rely only on synthetic `type:"compact"`.
   - Treat top-level `session` metadata as synthetic-only; real Claude lines should use file path, `sessionId`, `cwd`, `gitBranch`, `uuid`, and `parentUuid`.
4. Keep warnings actionable.
   - Unknown warnings should remain for truly unknown records.
   - Known real record types from Phase 8 fixtures should not warn.
5. Update parser tests from Plan 01 so they fail on old behavior and pass after repair.

## Acceptance Criteria

- Codex `function_call_output` under `response_item.payload` creates a result event on the matching call.
- Codex `custom_tool_call` / `custom_tool_call_output` appears as a structured tool call/result pair.
- Claude `tool_result` blocks pair with prior `tool_use.id`.
- Claude `thinking` is retained in replay-accessible data and is not silently dropped.
- Claude real compact summary records are preserved as a compact/system boundary.
- Every parser-created tool call has enough metadata for sync to write `tool_calls.message_ordinal`.
