# Plan 08-04: Wire sync-first refresh through ingest API, BFF, and right rail

## Goal

Make manual refresh mean "scan/reindex local files, then reload UI data" rather than only refetching stale DB rows.

## Tasks

1. Extend ingest sync API behavior from Plan 03:
   - `POST /api/v1/sources/:type/sync` accepts `force`.
   - Response includes `messagesInserted`, `sessionsInserted`, `sessionsUpdated`, `errors`, and whether force was used.
2. Update aggregate BFF route `app/api/sync/route.ts`.
   - Accept optional `force` from query/body.
   - Loop all source types and call ingest source sync.
   - Return per-source results with sanitized errors.
3. Add per-source BFF route `app/api/agent-tools/[tool]/sync/route.ts`.
   - Validate `[tool]` with `assertSourceToolId`.
   - Reject `all` or unsupported tools.
   - Forward `force` to ingest.
   - Use `fetchIngest()` and `sanitizeError()`.
4. Add client helper in `lib/agent-tools/client-hooks.tsx`.
   - Example shape: `syncToolSessions(toolId, { force?: boolean })`.
   - Use BFF endpoints only.
   - Keep `notifySessionsRefresh()` as the refetch broadcast.
5. Update refresh controls.
   - `components/shell/shell-header.tsx`: keep sync-first aggregate behavior; forward errors into a visible/logged state if already supported.
   - `AggregateSessionsRightRail`: call aggregate sync then `notifySessionsRefresh()`.
   - `SourceSessionsRightRail`: call per-tool sync then `sourceSessions.refetch()` or `notifySessionsRefresh()`.
   - Disable/spin refresh button while sync is running to avoid overlapping syncs.
6. Add tests.
   - BFF route validates tool ids and forwards `force`.
   - Client refresh helper calls `/api/agent-tools/codex/sync` before `/api/agent-tools/codex/sessions`.
   - Aggregate refresh calls `/api/sync` before refetch event.
   - Sync failure preserves current list and exposes error state.

## Verification

- `pnpm test:run tests/hooks/client-hooks.test.tsx tests/unit/bff/sync-route.test.ts`
- `pnpm typecheck`

## Acceptance Criteria

- Right rail refresh for a source triggers ingest sync for that source before session list refetch.
- Aggregate refresh triggers all-source sync before list refetch.
- Header refresh keeps sync-first behavior and supports the same force forwarding where applicable.
- No client code calls `http://localhost:8078` directly.
- Force sync can be invoked through BFF for post-parser-fix reindex verification.
