**Main entry point — parseClaudeSession(filePath, project) → Promise<ParseResult>:**
Same structure as parseOpenClawSession(). Key differences:
- Use `ClaudeJsonlLine` instead of `OpenClawJsonlLine`
- Track seen UUIDs in a `Set<string>` for streaming dedup (D-03): skip line if `parsed.uuid` already in set
- Track DAG nodes in a `Map<string, ClaudeDAGNode>` (D-01): each message registers its uuid→parentUuid relationship
- After all lines parsed, resolve DAG: iterate nodes, for each with `parentUuid`, link to parent's `sessionId`
- Track compact boundaries in `ClaudeCompactBoundary[]` (D-02): when line type is 'compact', record the affected UUIDs and mark those messages as truncated
- Process `parsed.session` metadata: if `session.type` is 'subagent' or 'fork', set `rootSessionId`/`parentSessionId`/`relationshipType` on the TraceSession

**Session context extraction — extractClaudeSessionContext(filePath, project) → SessionContext:**
- Claude JSONL filenames follow pattern `{uuid}.jsonl` in `~/.claude/sessions/`
- Extract UUID from filename using regex: `/([0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})/i`
- If no UUID found, use `path.basename(filePath, '.jsonl')`
- Source version: extract from first line's `session` object if present, otherwise `'unknown'`

**Message parsing — parseClaudeMessage(raw: ClaudeJsonlLine, ordinal: number, context: SessionContext, lineNum: number) → TraceMessage:**
- `id`: `${context.uuid}-${ordinal}`
- `role`: map `parsed.message.role` to MessageRole ('assistant' → 'assistant', 'user' → 'user', 'system' → 'system')
- `content`: if string, use directly; if array of content blocks, concatenate text blocks, extract tool_use blocks separately
- `sourceMetadata`: `{ sourceType: 'claude-code', sourceFile: context.filePath, sourceLine: lineNum, sourceVersion: context.sourceVersion || 'unknown' }`

**Tool call extraction — extractClaudeToolCalls(raw: ClaudeJsonlLine, context: SessionContext, lineNum: number) → TraceToolCall[]:**
- Scan `raw.message.content` array for blocks with type 'tool_use'
- Use `block.id` as tool call ID (D-11: this is the tool_use_id for result pairing in the assembler)
- Category inference: reuse the `inferToolCategory()` logic from openclaw.ts (copy the function or import it). Claude tools have names like 'Bash', 'Read', 'Write', 'Edit', 'Grep', 'Task', 'NotebookEdit', 'WebSearch', 'WebFetch', etc.
- Map categories: 'Bash'→Bash, 'Read'→Read, 'Write'/'Edit'/'NotebookEdit'→Edit, 'Grep'/'Glob'→Grep, 'Task'→Task, 'WebSearch'/'WebFetch'→Other, default→Other

**DAG resolution — resolveClaudeDAG(nodes: Map<string, ClaudeDAGNode>) → Map<string, { parentSessionId: string; relationshipType: string }>:**
- For each node with `parentUuid`, look up the parent node
- Map `parentNode.sessionId` as `parentSessionId`
- Determine `relationshipType`: 'fork' if parent and child share same session (common root), 'continuation' if child continues parent's session, 'subagent' if session.type indicates spawned agent

**Deduplication (D-03):**
- Maintain `const seenUuids = new Set<string>()` in the main parse loop
- Before processing: `if (seenUuids.has(parsed.uuid)) continue; seenUuids.add(parsed.uuid);`
- Add warning: `warnings.push(`Line ${lineNum}: Duplicate UUID ${parsed.uuid} — skipping`)`

**Compact handling (D-02):**
- When `parsed.type === 'compact'`: record compact boundary with affected UUIDs
- Continue processing — compact events produce a system-level TraceMessage with role 'system' and content describing what was compacted
- Set `is_truncated: true` on messages whose UUIDs appear in compact boundaries

**Error handling:**
- Catch JSON.parse errors per line → `errors.push({ line: lineNum, raw: line.substring(0, 200), error: err.message })`
- Never throw; always return a ParseResult
- If file doesn't exist: return ParseResult with error-session (status: 'error') and appropriate error entry

**Token usage accumulation:**
- Extract `parsed.message.usage.input_tokens` and `output_tokens`
- Accumulate totals for SessionMetrics

**Session construction:**
- `source: 'claude-code'`
- `startedAt`: first parsed timestamp; `endedAt`: last parsed timestamp
- `status`: endedAt exists ? 'idle' : 'active'
- `rootSessionId`, `parentSessionId`, `relationshipType`: from session metadata resolution
- `metrics.messageCount`, `metrics.userMessageCount`, `metrics.totalTokens`, `metrics.hasToolCalls`, `metrics.parserMalformedLines`, `metrics.isTruncated`

```typescript
/**
 * Parse a single Claude Code message line
 *
 * Helper function for testing and single-line parsing scenarios.
 * Per D-03: Does NOT track UUID dedup (single-line context).
 *
 * @param line - JSONL line string
 * @param context - Session context
 * @returns TraceMessage or null if parsing fails
 */
export function parseClaudeMessage(
  line: string,
  context: SessionContext
): TraceMessage | null {
  try {
    const parsed: ClaudeJsonlLine = JSON.parse(line);
    if (!parsed.message) return null;
    return parseMessage(parsed, 0, context, 0);
  } catch {
    return null;
  }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSONL file → parser | Untrusted file content (malformed JSON, unexpected types, injection attempts) |
| session UUID → DAG resolution | Cross-session references must be validated (orphaned parentUuids) |
| tool input JSON → TraceToolCall | Tool input is arbitrary JSON from agent logs — must be stored as string, never evaluated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Spoofing | claude.ts: UUID dedup | mitigate | UUID Set prevents duplicate message injection; first-write-wins strategy per D-03 |
| T-03-06 | Tampering | claude.ts: JSONL parsing | mitigate | try/catch per line with ParseError accumulation; malformed lines never abort full parse |
| T-03-07 | Information Disclosure | claude.ts: source_file paths | accept | File paths stored in SourceMetadata for debugging; Phase 6 adds API-level path restrictions |
| T-03-08 | Denial of Service | claude.ts: large JSONL files | accept | Line-by-line streaming via readline avoids memory exhaustion; very large sessions (>100K lines) may be slow but this is a local tool |
| T-03-09 | Elevation of Privilege | claude.ts: tool input | mitigate | Tool input stored as JSON.stringify(input) — never parsed, evaled, or executed; stored as opaque string in TraceToolCall.inputJson |
| T-03-10 | Spoofing | claude.ts: DAG resolution | mitigate | Orphaned parentUuid references logged as warnings; no cross-file traversal; parentSessionId validated to exist in same parse batch |
