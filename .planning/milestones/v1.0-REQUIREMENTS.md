# Requirements Archive: agent-tracing-dashboard v1.0

**Status:** ✅ SHIPPED 2026-05-12
**Milestone:** v1.0 MVP
**Original defined:** 2026-05-06

## v1 Requirements — Final Status

### Foundation

- [x] **FOUND-01**: 项目文档、导航文案和默认入口使用 agent-tracing-dashboard 语义，不再把产品定义为单一 OVAO/OpenClaw visual office。 — ✅ Delivered Phase 1
- [x] **FOUND-02**: 代码中定义统一 Trace Contract，包含 Source、Session、Turn、Message、ToolCall、SkillUse、Subagent、Activity、TokenUsage、Timing metadata。 — ✅ Delivered Phase 1
- [x] **FOUND-03**: 建立 OpenClaw、Claude Code、Codex fixture corpus，并为 canonical parser 输出建立黄金样例。 — ✅ Delivered Phase 1, expanded in Phase 8
- [x] **FOUND-04**: 保留现有 OpenClaw Gateway live overview 能力，避免改造期间丢失已完成的 Agent/KPI/Sessions/Cron/Skills/Activity 信息。 — ✅ Delivered Phase 1, Phase 4
- [x] **FOUND-05**: 前端提供 source-aware 空状态、错误状态和配置状态，能区分未安装、未配置、无 session、读取失败、解析失败。 — ✅ Delivered Phase 4

### Ingest Data Plane

- [x] **DATA-01**: 新增独立 Node/TypeScript ingest service，可启动 health/version/sources/events API。 — ✅ Delivered Phase 2
- [x] **DATA-02**: ingest service 使用 SQLite WAL/FTS5 存储本地索引。 — ✅ Delivered Phase 2, enhanced Phase 8
- [x] **DATA-03**: ingest service 支持 OpenClaw、Claude Code、Codex 的默认目录发现和 source health 状态。 — ✅ Delivered Phase 2, Phase 3
- [x] **DATA-04**: ingest service 支持 chokidar/Node 文件监听、debounce、periodic resync fallback、skip cache。 — ✅ Delivered Phase 6
- [x] **DATA-05**: ingest service 暴露 REST API：sources、sessions、session detail、turns、messages、tools、children、search、sync/resync。 — ✅ Delivered Phase 2, Phase 3, enhanced Phase 8
- [x] **DATA-06**: ingest service 暴露全局和单 session SSE。 — ✅ Delivered Phase 6
- [x] **DATA-07**: API 只能按已索引 session id 读取受控 source file，不接受客户端任意文件路径。 — ✅ Delivered Phase 6

### Source Parsers

- [x] **SRC-01**: OpenClaw parser 支持 session header、message、toolResult role、usage 字段归一化。 — ✅ Delivered Phase 2
- [x] **SRC-02**: Claude Code parser 支持 DAG/fork/continuation、queued command、streaming duplicate collapse、compact/system boundary 和 subagent mapping。 — ✅ Delivered Phase 3, repaired Phase 8
- [x] **SRC-03**: Codex parser 支持 session_meta、turn_context、response_item、event_msg、function_call/function_call_output、spawn_agent/wait/subagent notification。 — ✅ Delivered Phase 3, repaired Phase 8
- [x] **SRC-04**: 三个 parser 都输出 canonical Message、ToolCall、ToolResultEvent、SubagentLink 和 source metadata。 — ✅ Delivered Phase 3
- [x] **SRC-05**: parser 记录 termination_status、is_truncated、parser_malformed_lines、source_version、cwd/git_branch 等调试字段。 — ✅ Delivered Phase 3

### Turn Replay Model

- [x] **TURN-01**: ingest 层提供 turn-first read model。 — ✅ Delivered Phase 2, enhanced Phase 3
- [x] **TURN-02**: 每个 turn 保留 startedAt、endedAt、duration、token usage、model、failure/error 状态。 — ✅ Delivered Phase 3
- [x] **TURN-03**: Tool call 按 tool_use_id/call_id/toolCallId 精确配对结果。 — ✅ Delivered Phase 3, enhanced Phase 8
- [x] **TURN-04**: Skill 使用以独立 block 呈现。 — ✅ Delivered Phase 5
- [x] **TURN-05**: Subagent 调用支持 subagentSessionId，可 lazy 展开或打开完整子 session。 — ✅ Delivered Phase 5, enhanced Phase 9
- [x] **TURN-06**: System/compact/queued/interruption 等边界事件保留在数据模型中。 — ✅ Delivered Phase 5

### Frontend Architecture

- [x] **UI-01**: 前端采用 source-first 路由和 header switcher。 — ✅ Delivered Phase 4
- [x] **UI-02**: 实现 AgentToolProvider/registry/capability flags/UI profiles。 — ✅ Delivered Phase 4
- [x] **UI-03**: 保留 legacy redirects。 — ✅ Delivered Phase 4
- [x] **UI-04**: Session Explorer 支持 source、project/workspace、model、status、时间、搜索过滤。 — ✅ Delivered Phase 4
- [x] **UI-05**: 前端通过 trace API client/store/selectors 读取 ingest API。 — ✅ Delivered Phase 4

### OpenClaw Dashboard

- [x] **OPEN-01**: OpenClaw dashboard 保留并增强现有 overview。 — ✅ Delivered Phase 4
- [x] **OPEN-02**: OpenClaw live Gateway 数据和 ingest 历史 session 通过 session key/session id 做 best-effort link。 — ✅ Delivered Phase 6
- [x] **OPEN-03**: OpenClaw 无 Gateway 或 ingest 未启动时仍显示明确状态。 — ✅ Delivered Phase 6

### Replay UI

- [x] **REPLAY-01**: Session replay 页面按 turn 展示用户输入、assistant 响应、tool/skill/subagent/activity。 — ✅ Delivered Phase 5
- [x] **REPLAY-02**: Replay UI 支持长 session 虚拟化或 range pagination。 — ✅ Delivered Phase 5
- [x] **REPLAY-03**: Tool block 显示 tool name/category、input、result/status/error/duration，支持 copy。 — ✅ Delivered Phase 5, enhanced Phase 9
- [x] **REPLAY-04**: Subagent inline view 支持 lazy load 子 session。 — ✅ Delivered Phase 5
- [x] **REPLAY-05**: Replay 页面支持 in-session search、block filters、上一/下一 turn 导航。 — ✅ Delivered Phase 5
- [x] **REPLAY-06**: Replay 页面支持 copy message、copy tool、copy turn。 — ✅ Delivered Phase 5
- [x] **REPLAY-07**: Replay 页面展示 running/awaiting user/aborted/error/truncated/parser warning 状态。 — ✅ Delivered Phase 5

### Hardening

- [x] **HARD-01**: parser fixture tests 覆盖多种边界场景。 — ✅ Delivered Phase 6, expanded Phase 8
- [x] **HARD-02**: ingest API 和前端能处理长 session。 — ✅ Delivered Phase 6
- [x] **HARD-03**: 本地路径、session id、source roots 经过安全约束。 — ✅ Delivered Phase 6
- [x] **HARD-04**: 隐私默认值明确。 — ✅ Delivered Phase 6
- [x] **HARD-05**: 开发期启动流程能同时启动/连接 Next.js 和 ingest service。 — ✅ Delivered Phase 2, Phase 6
