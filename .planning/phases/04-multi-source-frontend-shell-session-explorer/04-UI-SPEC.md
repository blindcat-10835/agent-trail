> Visual and interaction contract for Multi-source Frontend Shell + Session Explorer.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, inline chip padding gap |
| sm | 8px | Compact element spacing, filter chip gaps |
| md | 16px | Default element spacing, section padding in cards |
| lg | 24px | Section padding, KPI tile padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Header height, major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions (layout dimensions, not spacing tokens):

| Dimension | Value | Usage |
|-----------|-------|-------|
| Sidebar width | 56px | Vertical icon nav rail — thin, icon-only |
| Right rail width | 360px | Session list + detail panel (fixed, not fluid) |
| Status bar height | 26px | Bottom status strip |
| Header height | 48px | Top bar with source switcher + controls |

**Grid contract:** The ShellFrame grid is `grid-rows-[48px_1fr_26px]` and content area is `grid-cols-[56px_minmax(0,_1fr)_360px]` when right rail is open, collapsing to `gridcols-[56px_minmax(0,_1fr)_0px]` when closed. This grid contract is the single source of truth for all shell dimensions.

**Content area padding:** Content within the main area (`children` slot of ShellFrame) uses `p-4` (16px) as default internal padding.

## Typography

| Role | Size | Weight | Line Height | Style |
|------|------|--------|-------------|-------|
| Label | 10px–11px | 600 (semibold) | 1.3 | Uppercase, letter-spacing: 0.12em–0.2em, `text-muted-foreground` |
| Body | 14px | 400 (normal) | 1.5 | `text-foreground` or `text-muted-foreground`, font-sans |
| Heading | 16px | 700 (bold) | 1.25 | `text-foreground`, tracking: -0.01em |
| Display | 30px | 700 (bold) | 1.2 | KPI values in stats bars, tabular-nums, `tracking-tight` |

**Monospace:** `font-mono` (JetBrains Mono) reserved for: model names, token counts, session IDs, code snippets, timestamps with `tabular-nums`, status codes.

**Font family hierarchy:**
- UI chrome (header, sidebar, status bar, nav): 11px semibold uppercase tracking-wide — Geist sans
- Body content (session labels, descriptions): 14px normal — Geist sans
- Technical data (model names, IDs, counts): 14px normal — JetBrains Mono monospace
- Headings (page titles, section labels): 16px bold — Geist sans

**Status badge sizing:** All status badges (LIVE, IDL, ABT, ERR, WARN) use 10px semibold uppercase tracking-[0.12em] for consistency with existing `sessions-table.tsx` pattern.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.14 0.008 160)` (`--background`) | Full-page background, main content area backdrop |
| Secondary (30%) | `oklch(0.185 0.008 160)` (`--card`) | Cards, sidebar nav background, header background, KPI tiles, filter bar, status bar |
| Accent (10%) | `oklch(0.8 0.17 75)` (`--accent`) | See reserved-for list below |
| Destructive | `oklch(0.704 0.191 22.216)` (`--destructive`) | Destructive actions, error badges, aborted status |

**Accent reserved exclusively for:**
1. Source switcher active tab indicator (underline/border)
2. Sidebar nav active state (left accent bar + text color)
3. Header bottom gradient line (`via-accent` in gradient)
4. Status bar top gradient line (`via-accent` in gradient)
5. Glow effects on active/focused elements (`hud-glow` box-shadow)
6. LIVE status indicator (pulsing dot + label color)
7. Interactive hover state borders (`hover:border-accent`)
8. Brand logo/diamond (header icon)
9. Section divider accent dots (small `bg-accent` squares in filter headers)
10. Focus rings on input/select elements (`focus:ring-accent`)

**NOT accent-reserved for:** section backgrounds, text body, card borders (use `--border`), secondary actions, info badges, idle status.

**Border hierarchy:**
- `--border`: `oklch(0.3 0.008 160)` — standard panel/row borders
- `--border-strong`: `oklch(0.4 0.01 160)` — emphasis borders (header bottom, active state)
- Gradient lines: `from-transparent via-accent to-transparent opacity-60` — decorative horizontal rules in header/footer

**Status colors (non-shadcn):**
- LIVE/Active: `oklch(0.76 0.17 145)` — green, used for live session indicators
- OFFLINE/Disconnected: `#6b7280` — gray, used in `status-indicator.tsx`
- CONNECTING: `#eab308` — yellow
- RECONNECTING: `#f97316` — orange
- ERROR: `#ef4444` — red

**Dark-first assumption:** The dark theme (`[data-theme="dark"]`) is the primary visual identity. Light theme (`:root`) is a shadcn fallback and not actively designed. All new components must look correct in dark mode; light mode is best-effort.

## Interaction Contract

### Source Switching
1. **Trigger:** User clicks source tab in header (OpenClaw / Claude Code / Codex).
2. **Behavior:** Navigate to same page under new tool segment (e.g., `/openclaw/dashboard` → `/codex/dashboard`). Preserve sub-route. Do NOT full-page reload — Next.js client-side navigation.
3. **Transition:** Instant (no animation). Previous tool's page content unmounts; new tool's content mounts. No crossfade.
4. **Loading:** Show skeleton placeholders while ingest data loads (per-tool, not global).
5. **Status feedback:** Source status dot in each tab (green=LIVE/SYNCED, gray=OFFLINE, yellow=SYNCING, red=ERROR).

### Right Rail
1. **Visibility:** Always visible by default (`rightRailOpen: true` in `useUIStore`). Persists across tool switches and page navigations within Phase 4.
2. **Toggle:** «/» button in header controls. Collapses to 0px width with CSS transition (`transition-[grid-template-columns] duration-200`).
3. **Content:** Left panel = session list for current tool (or aggregate on `/`). Clicking a session replaces right panel content with session detail.
4. **Detail panel:** Header (session label, model, status badge), KPI strip (tokens, cost, kind, created), message/event list. Closes via ✕ button or clicking different session.

### Session Explorer
1. **Data source:** BFF proxy → ingest API. NOT Gateway.
2. **Columns:** Label, Status, Model, Updated — base set. Per-tool profiles add/remove columns.
3. **Sorting:** Click column header to sort (ascending/descending/none). Default: updated descending (most recent first).
4. **Selection:** Single-select. Clicking a row selects it and opens detail in right rail. Clicking same row again deselects (closes detail).
5. **Expand:** Optional row expansion shows session metadata (tokens, cost, kind, last message) — pattern from existing `sessions-table.tsx`.
6. **Filtering:** Collapsible filter bar with chips for status, model, kind + search input. Filters from ingest query parameters (server-side filtering preferred where supported).

### Aggregate Landing Page
1. **Route:** `/` — root path.
2. **Content:** Session Explorer showing all sessions from all 3 tools merged and sorted by recency.
3. **Source badge:** Each row shows a source badge (OPENCLAW / CLAUDE:CODE / CODEX) differentiating entries.
4. **Stats bar:** Cross-source totals (all 3 tools combined).
5. **Drilldown:** Clicking a session navigates to `/{tool}/sessions/{sessionId}` for full replay context.

### Per-tool Dashboard Pages
- **OpenClaw (`/openclaw/dashboard`):** Preserve existing overview skeleton: KPI bar, agent card grid, sessions list, skills, cron, activity sections. Layout preserved; data disconnected from Gateway (empty placeholders).
- **Claude Code (`/claude-code/dashboard`):** Session summary stats from ingest: total sessions, active sessions, model breakdown, recent sessions list. No agent grid, no cron, no skills.
- **Codex (`/codex/dashboard`):** Same pattern as Claude Code: session summary stats + recent sessions list.

### Empty States
- **Aggregate (`/`):** HUD panel centered in main area. Heading: "NO SESSIONS INDEXED". Body: "START AN AGENT SESSION IN ANY SUPPORTED TOOL TO SEE IT APPEAR HERE."
- **Per-tool sessions:** HUD panel. Heading: "NO SESSIONS". Body: "ENSURE [TOOL] SESSIONS DIRECTORY IS CONFIGURED IN INGEST."
- **Per-tool dashboard:** Same pattern, adapted per tool.

### Error States
- **Ingest unreachable:** Header shows red ERROR indicator. Content area shows error panel. Heading: "INGEST UNREACHABLE". Body: "ENSURE THE INGEST SERVICE IS RUNNING. CHECK PNPM DEV FOR STATUS." Retry button available.
- **Source unconfigured:** Per-tool pages show error panel. Heading: "SRC OFFLINE". Body: "[TOOL] SESSION DIRECTORY NOT FOUND. CHECK INGEST CONFIG."
- **API error (generic):** Inline error on affected component. Heading: "ERR". Body: technical error message with retry option.

### Loading States
1. **Full page (initial load):** Skeleton grid matching target layout (shadcn `skeleton` components). Session table skeleton: 5 rows × 4 columns.
2. **Tool switch:** Content area only — skeleton while new tool data loads. Shell (header, sidebar, status bar) remains interactive.
3. **Session detail rail:** Loading spinner in center of panel during fetch. No skeleton.
4. **Filter change:** Brief spinner or disabled state while filtered results load. No skeleton flicker.

## Accessibility Baseline

### Keyboard Navigation
- Source switcher tabs: Arrow keys to navigate, Enter/Space to select
- Sidebar nav: Tab to enter, Arrow keys up/down, Enter to activate
- Session table: Tab to enter table, Arrow keys up/down to move between rows, Enter to select
- Right rail close: Escape key dismisses detail panel
- Filter bar: Tab to reach, Enter/Space to expand/collapse, Tab through filter chips
- Search input: Focus visible ring (`focus:ring-1 focus:ring-accent`)

### Screen Reader
- Source switcher tabs: `role="tab"`, `aria-selected`
- Sidebar nav items: `title` attribute for full label (e.g., `title="Sessions"` for "SES")
- Session table: `role="grid"` with `role="row"`, sortable column headers as `role="columnheader"` with `aria-sort`
- Status badges: `aria-label` for full status text
- Right rail: `aria-label="Session panel"`, close button: `aria-label="Close details"`
- Stats KPIs: `aria-label` with full label + value

### Visual
- Focus rings on all interactive elements: `focus:ring-1 focus:ring-accent`
- Minimum touch target: 40px × 40px for sidebar nav buttons, 28px × 28px for header controls
- Status indicators include both color AND text label (do not rely on color alone)
- Loading/error states are text+icon, not just visual

## Integration Points

### BFF API Proxy
- **Route pattern:** `/api/agent-tools/[tool]/sessions`, `/api/agent-tools/[tool]/sessions/[id]`, `/api/agent-tools/[tool]/sessions/[id]/turns`, `/api/agent-tools/[tool]/sessions/[id]/messages`
- **Data shape:** Canonical types from `types/trace.ts` — `TraceSession`, `TraceTurn`, `TraceMessage`, `TraceSourceMetadata`
- **Error shape:** `{ error: string, code: number }` — consistent across all proxy routes
- **Caching:** Next.js `fetch` with `next: { revalidate: 30 }` for session lists; `no-store` for session detail (to reflect fresh ingest data)

### Frontend Data Hooks
- `useToolSessions(toolId)` — fetches sessions for a tool, returns `{ sessions, loading, error, refetch }`
- `useSessionDetail(toolId, sessionId)` — fetches session detail + turns, returns `{ session, turns, loading, error }`
- `useAggregateSessions()` — fetches merged sessions from all tools, returns same shape
- `useSourceStatus(toolId)` — fetches source metadata (ingest status, session count)

### Zustand Stores
- `useUIStore` — preserves current `rightRailOpen`, `toggleRightRail`, `setRightRailOpen`. No changes needed.
- `useGatewayStore` — preserved for OpenClaw Overview only. NOT used by Session Explorer or other tool pages.
- New: `useToolStore` — stores currently selected `[tool]` + selected session ID. Minimal global state.

### ShellFrame Contract
- **Props:** `{ children: React.ReactNode }` (no tool prop — reads from context)
- **Must mount:** `AgentToolProvider` (wraps ShellFrame), `ShellHeader` (reads profile), `SidebarNav` (reads nav items), `ShellStatusBar` (reads connection state), optional `RightRail`
- **Grid contract (immutable):** `grid-rows-[48px_1fr_26px]`, content area with sidebar + main + right rail columns
- **Must NOT:** Make its own data fetches, contain Gateway-specific logic, hardcode any tool name
