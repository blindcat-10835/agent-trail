  switch (window) {
    case 'today': return `${column} >= datetime('now', 'start of day')`;
    case '7d': return `${column} >= datetime('now', '-7 days')`;

**1. GET /api/v1/overview/aggregates (DATA-101)**
- Query params: `source` (optional, one of openclaw/claude-code/codex), `window` (default '7d', one of today/7d/30d)
- Returns: `{ sessionCount, turnCount, projectCount, inputTokens, outputTokens, totalTokens }`
- SQL: `SELECT COUNT(*), COUNT(DISTINCT project), COALESCE(SUM(user_message_count), 0), COALESCE(SUM(total_input_tokens), 0), COALESCE(SUM(total_output_tokens), 0) FROM sessions` with optional source + date filters
- Per CONTEXT.md: `all` source scope = no source filter in query
- When `source` is invalid, return 400

**2. GET /api/v1/overview/top-models (DATA-102)**
- Query params: `source` (optional), `window` (default '7d'), `limit` (default 10, max 50), `sortBy` (default 'tokens', one of tokens/cost)
- Returns: `{ models: [{ name, sessionCount, inputTokens, outputTokens, totalTokens, sharePercent }] }`
- SQL: JOIN sessions with messages (where model IS NOT NULL), GROUP BY messages.model
- Compute sharePercent: `(model.totalTokens / totalAllTokens) * 100` rounded to 2 decimal places
- For cost sort: return `cost: null` per CONTEXT.md (no price data yet), but order by totalTokens as proxy

**3. GET /api/v1/overview/top-projects (DATA-103)**
- Query params: `source` (optional), `window` (default '7d'), `limit` (default 10, max 50)
- Returns: `{ projects: [{ project, sessionCount, turnCount, inputTokens, outputTokens, totalTokens, rankWeight }] }`
- SQL: GROUP BY sessions.project from sessions table
- rankWeight: `(project.totalTokens / totalAllTokens * 100)` rounded to 2 decimals

**4. GET /api/v1/overview/starred (DATA-104)**
- Query params: `source` (optional), `limit` (default 20, max 100)
- Returns: `{ starred: [{ id, name, source, project, status, startedAt, updatedAt, starredAt }] }`
- SQL: JOIN session_stars with sessions, ORDER BY starred_at DESC
- Use UPDATED_AT_EXPR for updatedAt

**5. GET /api/v1/overview/timeline (DATA-105)**
- Query params: `source` (optional), `limit` (default 50, max 200)
- Returns: `{ timeline: [{ id, source, eventType, eventTime, project, name, status, errorMessage }] }`
- SQL: UNION ALL of (a) session lifecycle events from sessions table (started_at, status) and (b) sync errors from sync_status (last_error IS NOT NULL)
- Per CONTEXT.md: no new activity_events table, built at query time from existing tables
- event_type values: 'session_started', 'session_completed', 'session_error', 'sync_error'

**6. GET /api/v1/overview/capabilities (DATA-106)**
- Returns: `{ capabilities: SOURCE_CAPABILITIES, sources: VALID_SOURCES }`
- Pure static response, no DB query

**7. GET /api/v1/overview/agents (OPEN-101)**
- Query params: `source` (required, must be one of valid sources)
- Returns: `{ agents: [{ name, sessionCount, toolCallCount, lastActiveAt, latestStatus }] }`
- SQL: Reuse pattern from `ingest/api/agents.ts` — GROUP BY agent_name
- This endpoint wraps the existing agents query for overview consumption

**8. GET /api/v1/overview/status (OPEN-103)**

- Ingest status from context/health
- Watcher status from context
- Gateway: return `status: 'disconnected'` as placeholder (gateway connectivity is existing functionality, per deferred ideas)
