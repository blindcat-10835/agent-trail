## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, copy button padding, status dot margin |
| sm | 8px | Compact element spacing, filter chip gaps, badge padding |
| md | 16px | Turn card internal padding, block indentation, section gap |
| lg | 24px | Turn card margin, right rail padding, search bar spacing |
| xl | 32px | Replay header to content gap, filter bar to turn list |
| 2xl | 48px | Page-level vertical rhythm, major section breaks |
| 3xl | 64px | Top/bottom page padding |

**Exceptions:** HUD clipped containers occasionally use 6px (gap-1.5), 10px (px-2.5), and 14px (gap-3.5) for tight visual density within border-clipped elements. These are established HUD patterns from Phase 4 and may be used for status indicators, clip-path badges, and header controls.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Micro Label | 10px | Semibold (600) | 1.2 | Uppercase section labels (e.g., "TOOLS", "MESSAGES"), KPI labels, status badges |
| Body | 12px | Regular (400) | 1.5 | Turn content, message text, tool input/output, metadata values |
| Heading | 14px | Semibold (600) | 1.3 | Turn number headers, block titles, right rail section headings |
| Display | 20px | Semibold (600) | 1.2 | Session name in replay header, breadcrumb current page |

**Monospace overrides:** Tool input JSON, session IDs, and code blocks use `font-mono` (JetBrains Mono) at 11px Regular (400) with line-height 1.6.

**Letter spacing:** Micro labels and status badges use `tracking-[0.2em]` to `tracking-[0.25em]` uppercase. This is an established HUD convention.

### Accent Reserved For (explicit list)

| Use | Implementation |
|-----|---------------|
| Interactive element borders on hover/focus | Nav items, filter chips, expand/collapse toggles, copy buttons |
| Active/selected state indicators | Current filter chip active, current turn indicator in timeline |
| Primary interactive text | Breadcrumb links ("Sessions"), "View Session" CTA, tool/skill/subagent names in blocks |
| Status glow effects | Pulse animations on active/running indicators, tool status dots |
| Header brand elements | Replay header accent line, breadcrumb separator |
| Copy confirmation flash | "Copied!" feedback text and brief border flash |
| Keyboard shortcut hints | ("j/k" in turn navigator) |
| Search match highlights | Inline search result highlights within message content |

**NOT reserved for:** Body text, card backgrounds, metadata labels (`--muted-foreground`), error states (`--destructive`), non-interactive decorative elements.

## Copywriting Contract

### Primary Actions

| Element | Copy |
|---------|------|
| Primary CTA (from Session Explorer) | View Session |
| Secondary CTA (within replay) | Copy Turn |
| Copy confirmation | Copied! (2000ms auto-dismiss) |
| Subagent CTA | Load Subagent |
| Subagent full session link | Open Full Session → |
| Expand all turns | Expand All |
| Collapse all turns | Collapse All |
| Retry action | RETRY |

### Empty States

| State | Heading | Body | Action |
|-------|---------|------|--------|
| No session selected | NO SESSION | Select a session from the explorer to view its turns. | — |
| Session has no turns | NO TURNS | This session contains no parsed turns. The file may be empty or still processing. | — |
| Search has no results | NO MATCHES | No turns match your search. Try different terms or clear filters. | CLEAR SEARCH |

### Error States

| State | Heading | Body | Action |
|-------|---------|------|--------|
| Fetch failed (network/server error) | ERR | Could not load session turns. | RETRY |
| Session not found (404) | NOT FOUND | Session data is not available. It may have been removed or the ID is invalid. | BACK TO SESSIONS |
| Subagent load failed | ERR | Could not load subagent turns. | RETRY / DISMISS |

### Loading States

| State | Visual |
|-------|--------|
| Initial page load | Skeleton cards (3 placeholder turn cards using `Skeleton` component), no text |
| Turn page fetch (pagination) | Skeleton rows at bottom of virtualized list, no text |
| Subagent lazy load | Spinner inside subagent block, no text |
| Search debounce | No loading indicator (results update in-place after 300ms debounce) |

### Destructive Actions

None in this phase. Copy actions are the only mutating operations and are non-destructive.

### State Store Shape

`stores/replay-store.ts` (Zustand, follows `useUIStore` pattern):

```typescript
interface ReplayState {
  // Scroll
  scrollPosition: number
  setScrollPosition: (pos: number) => void

  // Expand state (per turn, keyed by turnId)
  expandedTurns: Set<string>
  toggleTurn: (turnId: string) => void
  expandAll: () => void
  collapseAll: () => void

  // Filter (multi-select, persisted in URL search params)
  activeFilters: Set<'user' | 'assistant' | 'tools' | 'skills' | 'subagents' | 'system'>
  toggleFilter: (filter: string) => void
  setFilters: (filters: string[]) => void

  // Search
  searchQuery: string
  setSearchQuery: (query: string) => void
  searchMatches: { turnId: string; matchCount: number }[]
  currentMatchIndex: number
  setCurrentMatchIndex: (index: number) => void
}
```

## Interaction Patterns

### Turn Card Expand/Collapse

- Short sessions (≤10 turns): all expanded on first load
- Long sessions (>10 turns): collapsed by default, "Expand All" / "Collapse All" toggle visible
- Per-turn expand state persists during session view (in Zustand store, not URL)
- Expand animation: `max-height` transition (200ms ease-out) or CSS `@starting-style` with `interpolate-size: allow-keywords`
- Collapsed turn shows: turn number, user message first 2 lines (truncated), tool/skill count badges
- Clicking anywhere on collapsed turn header expands it; clicking an expanded turn header collapses it

### Tool Block

- Always rendered as inline sub-card between user and assistant messages within a turn
- Collapsed state: icon (lucide `Wrench`), tool name (bold), category badge, status dot (green=success, red=error, yellow=pending), duration
- Expanded state: all of collapsed + syntax-highlighted input JSON (monospace block), result event stream with timestamps
- Copy button on tool block copies the full tool call (name + input + result) as structured text
- Tool input JSON: syntax highlighted with a lightweight highlighter (or plain monospace with indentation), collapsible if >10 lines
- Status dot uses `animate-pulse` for `pending` state

### Skill Block

- Same visual hierarchy as tool block but distinct icon (lucide `Sparkles`) and color treatment
- Collapsed: skill name, truncated input summary (first 80 chars), status badge
- Expanded: full input description, result/status
- No copy action needed on skill blocks per current requirements (can be added later)

### Subagent Block

- Collapsed: lucide `Bot` icon, subagent session ID (truncated monospace), "Load Subagent" button
- On load: fetches child session turns via BFF proxy, renders nested mini-timeline (depth 2 cap)
- If child session exceeds depth 2, show "Max nesting depth reached" message instead of further recursion
- "Open Full Session" link button navigates to `/openclaw/sessions/[childSessionId]` (full-page replace or new tab)
- Loading state: spinner only, no text
- Error state: "ERR / Could not load subagent turns. RETRY / DISMISS"

### Search

- Search bar at top of replay area, below header
- Debounced 300ms before executing search
- Searches: message content (user + assistant) AND tool names
- Match highlights: inline yellow/green background highlight on matching text within visible turn cards
- Match counter: "3 of 12 matches" displayed in search bar
- Jump-to-match: Enter jumps to next match, Shift+Enter to previous; scroll into view with smooth animation

### Filter Chips

- Toggle bar between search bar and turn timeline
- Multi-select: clicking a chip toggles it on/off
- Active chip: `bg-accent/15 border-accent text-accent`
- Inactive chip: `bg-card border-border text-muted-foreground`
- Persisted in URL search params: `?filter=user,tools`
- "All" chip: special behavior — selecting it deselects all others; selecting any other chip deselects "All"

### Turn Navigation

- Prev/Next buttons: fixed position or in header area
- Keyboard shortcuts: `j` (next) and `k` (previous), scoped to replay page (not global)
- `/` focuses search bar
- `Enter` expands/collapses currently focused turn
- Jump-to-turn input: number input + "Go" button that scrolls to that turn index
- Smooth scroll animation (300ms) when navigating between turns
- Current turn highlighted with accent-left-border or glow

### Copy Actions

- Copy Turn: copies full turn (user message + assistant response) as markdown-formatted text
- Copy Message: per-message copy button on hover, copies raw content
- Copy Tool: per-tool-block copy button, copies tool name + input JSON + result text
- All copy actions show "Copied!" text feedback for 2000ms, auto-dismiss
- Uses `navigator.clipboard.writeText()` with fallback (no external library needed)

### Session Status Display

- Prominent status bar/badge at top of replay area (below header, above turn list)
- Status values and their visual treatment:

| Status | Label | Color | Visual |
|--------|-------|-------|--------|
| `active` | LIVE | `oklch(0.76 0.17 145)` green | Pulse animation dot + uppercase label |
| `idle` | IDLE | `--muted-foreground` | Static, muted |
| `aborted` | ABORTED | `--destructive` | Red, with warning icon |
| `error` | ERROR | `--destructive` | Red, with error icon |
| `running` | RUNNING | `--accent` | Pulse animation + accent color |
| `awaiting-user` | AWAITING USER | `oklch(0.76 0.17 75)` yellow | Pulse animation |
| `truncated` | TRUNCATED | `--muted-foreground` | Warning badge with tooltip explaining truncation |
| `parser-warning` | PARSE WARNINGS | `oklch(0.76 0.17 55)` amber | Warning badge with count |

### Right Rail (Replay Mode)

- Toggle: collapsed by default (user expands via `»` button in header or dedicated toggle)
- Content sections:
  1. Session metadata: model, start/end time, duration, total tokens, source path, project
  2. Turn index: compact numbered list of turns; clicking jumps to that turn; current turn highlighted
- Metadata section follows existing `sessions-detail-rail.tsx` KPI strip pattern (4-column grid, uppercase micro labels)

### Virtualization

- Library: `@tanstack/react-virtual`
- Threshold: virtualize when session has >15 turns; below that, render all turns directly
- Uses `useVirtualizer` with dynamic item sizing (turns have variable heights depending on content)
- Pre-fetch: when scroll position is within 5 items of the end, fetch next page (50 turns) via BFF proxy
- Cache key: `sessionId + activeFilters + searchQuery`
- Scroll position: stored in `useReplayStore`, keyed by sessionId; restored on back-navigation; reset on session change or filter change
- No scroll/cache corruption when filters or sessions change (force remeasure on data change)

### Keyboard Shortcuts

| Key | Action | Scope |
|-----|--------|-------|
| `j` / `↓` | Next turn | Replay page |
| `k` / `↑` | Previous turn | Replay page |
| `/` | Focus search bar | Replay page |
| `Enter` | Expand/collapse focused turn | When turn card is focused |
| `Escape` | Blur search, collapse all | Replay page |
| `Shift+Enter` | Previous search match | When search is active |
