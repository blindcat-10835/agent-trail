## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env var → source path | Untrusted environment variables (CLAUDE_SESSIONS_PATH, CODEX_SESSIONS_PATH) determine filesystem paths |
| filesystem → parser types | JSONL file content is untrusted input; types define the expected shape but runtime validation is needed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | sources.ts: env var path | mitigate | Default to `os.homedir()` when env var is empty/unset; path traversal not mitigated here (Phase 6 adds API path constraints per DATA-07) |
| T-03-02 | Information Disclosure | types.ts: source_file fields | accept | Source file paths are stored for debugging; no PII in parser types; Phase 6 hardens API access |
| T-03-03 | Denial of Service | sources.ts: directory enumeration | accept | Large directories could be slow but this is a local tool scanning local session dirs; Phase 6 adds chokidar-based incremental sync |
| T-03-04 | Spoofing | types.ts: JSONL field types | mitigate | TypeScript interfaces provide compile-time shape checking; runtime validation against actual JSONL content is done in parser implementations (Plans 02, 03) |
</threat_model>
