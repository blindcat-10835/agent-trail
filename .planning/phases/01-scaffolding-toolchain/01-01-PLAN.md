- Which style would you like to use? → **New York**（现在是默认选项）

- Where is your global CSS file? → **app/globals.css**（默认应该已检测到）
- Would you like to use CSS variables for theming? → **Yes**（默认）
- Configure the import alias for components? → **@/components**（默认）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 开发环境 → 生产构建 | 本地开发工具链配置可能影响生产构建结果 |
| shadcn/ui CLI → 文件系统 | CLI 自动修改 globals.css 和创建组件文件 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | shadcn/ui CLI | accept | 使用官方 shadcn@latest 包，pnpm lockfile 验证包完整性 |
| T-01-02 | Tampering | globals.css | accept | CSS 变量为静态 HSL 值，Phase 2 定制后仍为设计令牌，无用户输入 |
| T-01-03 | Information Disclosure | build output | accept | 构建产物包含标准 Next.js 元数据，无敏感信息 |
| T-01-04 | Denial of Service | 开发服务器 | accept | 本地开发环境，无生产可用性影响 |

**注**: Phase 1 为脚手架配置阶段，不涉及用户输入、认证、会话管理等安全敏感功能。主要威胁是供应链欺骗（通过官方包和 lockfile 缓解）。
</threat_model>
