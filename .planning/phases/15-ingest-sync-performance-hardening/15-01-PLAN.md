  <action>
Create a small scheduler module that owns active and queued sync state.

Required API shape:

```ts
export type SyncReason = 'startup-warmup' | 'background' | 'watcher' | 'periodic' | 'manual'

export interface SyncSchedulerStatus {
  active: boolean
  activeReason: SyncReason | null
  activeScope: string | null
  queued: boolean
  queuedReasons: SyncReason[]
  startedAt: string | null
  lastCompletedAt: string | null
  lastDurationMs: number | null
  lastError: string | null
}
```

The scheduler must support:

- enqueue full-source work by source type
- enqueue path work by source type plus paths
- run only one sync at a time
- coalesce repeated requests while active
- expose status for health/debug
- record errors without dropping queued work

Keep implementation simple: a single in-process queue is enough. Do not introduce external dependencies.
  </action>
  <acceptance_criteria>
    - Scheduler tests prove two rapid enqueue calls do not run concurrently.
    - Scheduler tests prove a queued request runs after the active request completes.
    - Scheduler status reports active, queued, reason, duration, and errors.
  </acceptance_criteria>
</task>
