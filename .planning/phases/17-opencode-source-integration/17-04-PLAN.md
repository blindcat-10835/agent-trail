# Plan 17-04: Overview/Cost Integration and Documentation

Wave 4 — depends on sync being operational (Plan 17-03).

## Goal

Integrate opencode data into overview aggregates and cost display, ensure all frontend views work correctly, and update documentation.

## Tasks

### T-01: Overview API opencode integration

**File:** `ingest/api/overview.ts`

1. `VALID_SOURCES` already includes `'opencode'` from Plan 17-01
2. Verify aggregates query works with `source = 'opencode'`
3. Verify `source = 'all'` includes opencode sessions
4. Verify top-models returns opencode models with `providerID/modelID` format
5. Verify top-projects returns opencode projects
6. Verify timeline returns opencode session events

### T-02: Source-reported cost integration

**Files:** `ingest/api/overview.ts`, `ingest/api/sessions.ts`, `types/overview.ts`

1. When building overview aggregates, read `source_cost_usd` for opencode sessions
2. For sessions with `source = 'opencode'` and `source_cost_usd IS NOT NULL`: use reported cost instead of pricing registry estimate
3. For sessions with `cost_pricing_status = 'reported_zero'`: include as `$0.00` (exact, not estimated)
4. For `source = 'all'`: mix reported and estimated costs in the total
5. Set `pricingStatus` to `'priced'` for source-reported, existing logic for others
6. Update `PricingStatus` type to include `'reported'` and `'reported_zero'` values if needed

### T-03: Frontend cost display — ~ prefix convention

**Files:** `components/overview/kpi-hero.tsx`, `components/overview/top-models-table.tsx`, `components/overview/top-projects-table.tsx`

1. Verify existing `~` prefix logic for partial pricing works with new `reported` status
2. Source-reported cost: display without `~` prefix (e.g., `$1.23`)
3. Pricing registry estimate: display with `~` prefix (e.g., `~$1.23`)
4. `reported_zero`: display as `$0.00` without `~`

### T-04: Session detail and turn replay verification

**Files:** `app/(tool-shell)/[tool]/sessions/`, replay components

1. Verify session detail page renders opencode session header correctly
2. Verify turn replay shows user text, assistant text, reasoning, tool calls
3. Verify tool call rendering shows correct category icons
4. Verify step-start/step-finish system events render properly
5. Verify patch tool calls show file list
6. Verify subagent link navigation works
7. No new components needed — reuse existing shared replay components

### T-05: Documentation updates

**Files:** `docs/CONFIGURATION.md`, `docs/API.md`, `docs/db-schema.md`

1. `docs/CONFIGURATION.md`: add `OPENCODE_DB_PATH` env var documentation
2. `docs/API.md`: add opencode examples to source endpoints
3. `docs/db-schema.md`: document new cost columns and updated CHECK constraints
4. Update any "three sources" references to "four sources"

### T-06: Update ERRORS_LEARNED.md with pitfalls discovered

**File:** `ERRORS_LEARNED.md`

1. Document: SQLite CHECK constraint requires rebuild-table migration (not ALTER)
2. Document: `SQLITE_BUSY` handling pattern for concurrent DB access
3. Document: Token double-counting risk between session totals and step-finish
4. Document: `cost = 0` ambiguity — distinguish reported_zero from unknown

- [ ] Top models returns opencode models with correct provider/model format

- [ ] `reported_zero` displays as `$0.00` without `~`
