# Plan 17-03: Sync Integration and BFF/Frontend Registry

## Goal

Wire the opencode parser into the sync engine, add source discovery, create BFF adapter and frontend tool definition, so that `/opencode/sessions`, `/opencode/dashboard`, etc. work end-to-end.

## Tasks

### T-01: OpenCode source discovery

**File:** `ingest/sync/sources.ts`

1. Add `discoverOpencodeSources(dirs?: string[]): Promise<DiscoveredSource[]>`
2. Resolve DB path from env/config/default
3. If path exists and is readable, open readonly, validate schema (4 tables exist), count sessions
4. Return `DiscoveredSource { type: 'opencode', path, sessionCount, error? }`
5. Handle: DB not found â†’ `error`, schema mismatch â†’ `error`, locked â†’ `error` with warning
6. Update `getSourceConfig()` dispatch to handle `'opencode'`

### T-02: OpenCode sync handler

**File:** `ingest/sync/index.ts`

1. Add `syncOpencodeSource(opts: SyncSourceOptions): Promise<SyncResult>`
2. Open DB readonly, query all sessions
3. For each session: compute skip key from `time_updated + message count + part count`
4. If skip key matches existing `file_hash` in canonical DB â†’ skip
5. Otherwise: call `parseOpencodeSession(dbPath, rawSessionId, projectOverride)`
6. Set `file_path` to `dbPath#rawSessionId` for traceability
7. Call `writeSessionToDatabase(parseResult, db, filePath, { force: false })`
8. Handle parent session relationships: if `parent_id` exists, set `parentSessionId = 'opencode:${parent_id}'`
9. Close DB in finally
10. Update `syncSource()` dispatch to handle `'opencode'`
11. Update `parseFullCandidate()` dispatch to handle `'opencode'`
12. No incremental parse support for opencode in v1

### T-03: OpenCode discovery API endpoint
