## Goal

Implement the SQLite readonly parser that reads opencode sessions from `opencode.db` and produces canonical `ParseResult` objects.

## Tasks

### T-01: Create opencode parser with session reader

**File:** New `ingest/parser/opencode.ts`

1. Export `parseOpencodeSession(dbPath: string, rawSessionId: string, projectOverride?: string): Promise<ParseResult>`
2. Open DB readonly: `new Database(dbPath, { readonly: true, fileMustExist: true })`
3. Schema guard: query `sqlite_master` for `session`, `message`, `part`, `project` tables
4. Query session row + join project for worktree
5. Query message rows ordered by `time_created, id`
6. Query part rows ordered by `time_created, id` grouped by message
7. Map to canonical:
   - Session: `id = 'opencode:${session.id}'`, `source = 'opencode'`, token columns from session, cost from `session.cost`
   - Turn boundary: user message starts turn, subsequent assistant messages + parts belong to that turn
   - `tool` parts â†’ `TraceToolCall` with category mapping (bashâ†’Bash, readâ†’Read, edit/writeâ†’Edit, grep/globâ†’Grep, taskâ†’Task)
   - `reasoning` parts â†’ `TraceThinkingBlock`
   - `patch` parts â†’ `TraceToolCall` category Edit, tool name "patch", files array as inputJson
   - `step-start`/`step-finish` â†’ `TraceSystemEvent`
   - `subtask` parts â†’ `TraceSubagentLink` if session ref, else `TraceSystemEvent`
   - `file` parts â†’ attachment placeholder in message content
8. Handle `SQLITE_BUSY`: catch, retry 3x with 100ms delay, then throw descriptive error
9. Close DB in finally block
10. Never read `auth.json`

### T-02: Implement skip cache key generation

**File:** `ingest/parser/opencode.ts` (within same file)

1. Export `computeOpencodeSkipKey(session: OpencodeSessionRow, messageCount: number, partCount: number): string`
2. Build composite key: `opencode:${session.id}:${session.time_updated}:${messageCount}:${partCount}`
3. Hash with SHA-256 for consistency with existing skip cache format
