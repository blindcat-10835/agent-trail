<objective>
Replace whole-session delete/reinsert behavior on safe append deltas with idempotent append/upsert writes for messages, tool calls, result events, subagent links, and turn boundary data.
</objective>

<must_haves>
- Existing `writeSessionToDatabase()` remains the full replacement fallback.
- New append writer is used only for `IncrementalParseDelta` from a safe cursor decision.
- Repeating the same append delta does not duplicate messages, tool calls, result events, subagent links, or turn data.
- Cursor state is updated in the same transaction as append writes, after all rows are durable.
- No unbounded SQLite batching is introduced.
</must_haves>

<threat_model>
T-16-04: Append upserts can silently duplicate events if uniqueness is incomplete. Mitigation: add deterministic unique indexes or deterministic ids for appended derived rows and assert repeated sync idempotency in tests.
T-16-05: Updating cursor before writes commit can drop data after a crash. Mitigation: append rows and cursor update must share one SQLite transaction.
</threat_model>
