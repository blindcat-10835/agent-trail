<objective>
Add Claude and Codex append parsers that consume only newly appended complete JSONL lines when the Plan 01 cursor decision proves the file is append-only safe.
</objective>

<must_haves>
- Existing `parseClaudeSession()` and `parseCodexSession()` remain the full fallback parsers.
- Incremental parser APIs parse from byte offsets and do not read historical lines before the cursor offset.
- Parser deltas preserve message ordinal, turn index, model, tool-call, result-event, subagent-link, cwd, git branch, and token metadata needed by the writer.
- Unsafe or ambiguous parser state triggers full reparse instead of partial writes.
- No unbounded parser concurrency is introduced.
</must_haves>

<threat_model>
T-16-02: Offset parsing can lose cross-line parser context, especially tool results that arrive after the tool call. Mitigation: pass explicit incremental context from cursor/database state and fall back to full parse when required context is missing.
T-16-03: Malformed appended JSONL could advance the cursor and hide data loss. Mitigation: advance cursor only through complete lines that have been parsed and committed; record parser errors in the delta and fallback when errors make state unsafe.
</threat_model>
