# Activity Data Layer — Types, Log Parsing, and Environment Helpers

## Objective

Build the data layer for Activity Console: define Activity types, implement log parsing functions for cron runs and config audit logs, and create environment variable validation helpers. This layer will be consumed by API routes (Plan 02) and the Activity Console page (Plan 03).

**Purpose**: Establish a type-safe, filesystem-based data fetching system that reads cron execution records and config audit logs from OpenClaw's data directory, providing structured log entries for the Activity Console.

**Output**: Type definitions, log parsing utilities, and environment helpers that enable reading activity data from the filesystem.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-01 | Spoofing | API route | accept | No authentication required for local filesystem access (localhost only) |
| T-06-02 | Tampering | Filesystem | accept | Read-only operations (fs.readFileSync), no write/delete |
| T-06-03 | Repudiation | N/A | N/A | N/A |
