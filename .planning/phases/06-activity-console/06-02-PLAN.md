# Activity API Route — Fetch Logs from Filesystem

## Objective

Create the `/api/logs` API route that reads cron execution records and config audit logs from the filesystem, parses them using the log utilities from Plan 01, merges and sorts the entries, computes summary statistics, and returns the data as JSON.

**Purpose**: Provide a server-side API endpoint for the Activity Console page to fetch structured log data. The route uses filesystem access (not Gateway RPC) to read OpenClaw's log files directly from disk.

**Output**: A working API route at `/api/logs` that returns `{ entries: LogEntry[], summary: LogSummary }`.
