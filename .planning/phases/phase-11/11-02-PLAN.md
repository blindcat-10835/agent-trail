   ```typescript
   }
   ```
   Thread `railScope` through to `AggregateSessionsRightRail` and `SourceSessionsRightRail`, then to `SessionsRailContent`.

2. **Add scope-based session filtering** in `SessionsRailContent`:
   - Apply scope filter BEFORE the existing filter logic (search, source filter, group mode):
     ```typescript
     // Scope pre-filter
     let scopeFiltered = sessions
     if (railScope === 'starred') {
       scopeFiltered = sessions.filter((s) => starredIds.has(s.id))
     } else if (railScope === 'live') {
       scopeFiltered = sessions.filter((s) => s.status === 'active')
     }
     // 'recent' = no filter (default, shows all sessions sorted by updated_at)
     ```
   - Then apply existing search, source filter, group mode filters on `scopeFiltered` instead of raw `sessions`.
   - Show scope-specific count in the header. Replace the current "X indexed" with:
     - recent: `{count} indexed` (existing behavior)
     - starred: `{filteredCount} starred`
     - live: `{filteredCount} active`

3. **Add source-color spines** to `SessionRailRow`:
   - Define a source-color mapping constant (at the top of the file or near the component):
     ```typescript
     const SOURCE_SPINE_COLORS: Record<string, string> = {
       openclaw: 'border-l-[3px] border-l-[oklch(0.76_0.17_145)]',
       'claude-code': 'border-l-[3px] border-l-[oklch(0.8_0.17_75)]',
       codex: 'border-l-[3px] border-l-[oklch(0.76_0.17_200)]',
     }
     ```
   - Apply the spine class to the `SessionRailRow` outer `<div>`:
     ```tsx
     <div
       role="button"
       tabIndex={0}
       onClick={onSelect}
       className={cn(
         'grid w-full gap-1 px-3 py-2.5 text-left transition-colors hover:bg-accent/5',
         active && 'bg-accent/10 text-accent',
         SOURCE_SPINE_COLORS[session.source] || 'border-l-[3px] border-l-border',
       )}
     >
     ```
   - The `session.source` field (`TraceSource = 'openclaw' | 'claude-code' | 'codex'`) maps directly to the spine color keys.

4. **Handle scope switching correctly** — when the user switches scope:
   - Clear `selectedSessionId` if the selected session no longer exists in the new scope's filtered data (per research Pitfall 5). This prevents stale highlight.
   - The simplest approach: in `SessionsRailContent`, if `selectedSessionId` is set and the session is not in `filteredSessions`, don't highlight anything (the parent component can handle clearing).

5. **Starred scope special case** — when in 'starred' scope, the starred-only filter toggle should be disabled (it's redundant). Hide the starred filter in the dropdown when `railScope === 'starred'`.

6. **IMPORTANT**: Do NOT change the `<div role="button">` pattern on `SessionRailRow` — this avoids the nested-button anti-pattern (ERRORS_LEARNED.md EL-001).
