# Plan 08-01: Real-shape fixture corpus and opt-in local session harness

## Goal

Create privacy-safe test data that reflects actual Claude Code and Codex logs before changing parser behavior. This plan prevents another fixture-vs-real-data regression.

## Tasks

1. Add `tests/fixtures/real-shape/README.md` documenting fixture rules:
   - Use real local JSONL structure, but redact prompt text, tool output, absolute paths, filenames, and secrets.
   - Preserve envelope fields (`type`, `payload`, `uuid`, `parentUuid`, `call_id`, `tool_use_id`, timestamps) needed for parser behavior.
   - Do not commit complete local session files.
2. Add minimal committed fixtures:
   - Claude assistant `tool_use` plus user `tool_result` with matching `tool_use_id`.
   - Claude assistant `thinking` block interleaved with text.
   - Claude `isCompactSummary` / file-history-snapshot style compact boundary.
   - Codex `response_item.payload.type === "function_call_output"` with `output`.
   - Codex `custom_tool_call` / `custom_tool_call_output`.
   - Codex `reasoning` and `web_search_call` records that should not produce unknown warning spam.
3. Add `tests/fixtures/parser-regression/real-shape.test.ts` using these fixtures. Assertions should be structural, not full snapshots:
   - Parser does not throw.
   - Known real payload types do not produce unknown-type warnings.
   - Tool calls/results pair by id.
   - Thinking/compact records are retained in the chosen canonical representation.
4. Add `.local/real-session-corpus.example.json` with fake paths/ids and schema comments.
5. Update `.gitignore` to ignore `.local/real-session-corpus.json`.
6. Add `tests/local/real-session-corpus.test.ts`:
   - Skip unless `RUN_REAL_SESSION_TESTS=1`.
   - Load `.local/real-session-corpus.json` if present.
   - If manifest is absent, skip with a clear message, not failure.
   - Assert structural invariants only: parser does not crash, message ids are non-empty after sync, tool calls/results exist for tagged sessions, warnings are bounded.
7. Add `package.json` script:
   - `test:real-sessions`: `vitest run tests/local/real-session-corpus.test.ts`

## Verification

- `pnpm test:run tests/fixtures/parser-regression/real-shape.test.ts`
- `RUN_REAL_SESSION_TESTS=1 pnpm test:real-sessions` on this machine if local manifest is configured.
- `git status --short` must not show raw local session files.

## Acceptance Criteria

- Real-shape committed fixtures exist for the known Claude/Codex formats from the 2026-05-08 investigation.
- Local real-session corpus is opt-in, gitignored, and safe to skip.
- The plan explicitly covers target categories: Claude `606dac00-...`, Claude `effac644-...`, Codex function/custom tool sessions, and Claude subagent directory sessions through local corpus tags.
- No full local prompt/tool output is committed.
