# Quick Task 260508-18t: Fix session update times all showing same value

**Date:** 2026-05-07
**Commit:** 33502dd

## Summary

Fixed the Session Explorer table's "UPDATED" column which was showing identical update times across all sessions.

**Root Cause:** `renderCellValue()` in `session-explorer-table.tsx:108` used `session.endedAt || session.startedAt` instead of the API-computed `session.updatedAt` field. Sessions synced in the same batch share similar `endedAt` values, causing all rows to display the same relative time.

**Fix:** Changed `fmtAgo(session.endedAt || session.startedAt)` → `fmtAgo(session.updatedAt ?? null)` (1 line). This uses the server-side derived `updatedAt` field computed via SQL `MAX(COALESCE(ended_at, started_at, last_sync_at, file_mtime))`.

**Verification:**
- `pnpm typecheck` passes
- `grep` confirms `endedAt || startedAt` pattern no longer exists in the file
- `grep` confirms `session.updatedAt` is now used at line 108
