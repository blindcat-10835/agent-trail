<objective>
Fix session update time display showing identical values across all sessions in the Session Explorer table.

**Purpose:** The frontend `updatedAt` column was rendering `session.endedAt || session.startedAt` instead of the API-computed `session.updatedAt` field. Sessions synced in the same batch share similar `endedAt` values, causing all rows to display identical update times. The API correctly derives a freshest-timestamp `updated_at` via SQL (`MAX(COALESCE(ended_at, ''), COALESCE(started_at, ''), COALESCE(last_sync_at, ''), COALESCE(file_mtime, ''))`) and maps it to `updatedAt` — but the frontend table ignored this field entirely.

**Output:** 1-line fix in `session-explorer-table.tsx` — verified via `pnpm typecheck` and visual inspection.
</objective>
