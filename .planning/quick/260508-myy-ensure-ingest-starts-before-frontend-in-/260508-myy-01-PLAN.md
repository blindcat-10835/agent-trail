Create TWO new files:

**File 1: `stores/ingest-health-store.ts`** — Zustand store for ingest health state.

Follow the existing Zustand pattern from `stores/ui-store.ts` (import `create` from 'zustand', define interface, export hook).

Interface:
```typescript
interface IngestHealthState {
  status: 'checking' | 'connected' | 'timeout'
  retry: () => void
  setConnected: () => void
  setTimeout: () => void
}
```

- `status` starts as `'checking'`
- `setConnected()` sets status to `'connected'`
- `setTimeout()` sets status to `'timeout'`
- `retry()` resets status to `'checking'`

**File 2: `components/hud/ingest-health-overlay.tsx`** — Client component with full-screen overlay.

Component behavior (per decisions D-02 and D-03):
- `'use client'` directive at top
- On mount: begin polling `http://localhost:8078/health` every 2 seconds
- Track elapsed time with a ref; after 30 seconds without a healthy response → call `setTimeout()`
- **status === 'checking'**: Render full-screen overlay (fixed inset-0, z-40, bg-black/70 backdrop-blur-sm) with centered content: lucide `Loader2` icon with `animate-spin` class + text "Connecting to data service..."
- **status === 'connected'**: Render `null` (overlay hidden)
- **status === 'timeout'**: Render same overlay but with lucide `AlertTriangle` icon + error text "Unable to connect to data service" + a "Retry" button that calls `retry()` (resets to checking, restarts polling)
- If status changes from 'connected' back to 'checking' (ingest goes down after initial connection), show overlay again — this satisfies D-02 "any API call to ingest fails, not just initial load"
- Clean up interval on unmount
- Use semantic Tailwind v4 tokens: `text-foreground`, `bg-background`, `text-muted-foreground`
- Use `@/lib/utils` `cn()` if combining classes
- Use lucide-react icons: `Loader2` (spinner), `AlertTriangle` (error state)

Health check response handling:
- Fetch `http://localhost:8078/health`
- If response.ok and `data.status === 'ok'` → healthy (call `setConnected()`)
- Otherwise → not healthy (keep checking)
- Fetch errors (network down) → keep checking, continue polling

Port configurability note: The component currently uses a hardcoded `http://localhost:8078/health` URL. This is acceptable for a dev-focused quick task. If needed later, it can be made configurable via `NEXT_PUBLIC_INGEST_URL`.

DO NOT:
- Do NOT import from `@/lib/agent-tools/client-hooks` — this overlay is agent-tool-agnostic and must work at root layout level without AgentToolProvider context
- Do NOT add a dependency to package.json (use existing lucide-react and zustand)
- Do NOT use inline styles — use Tailwind classes consistent with the project's HUD/cyberpunk aesthetic
