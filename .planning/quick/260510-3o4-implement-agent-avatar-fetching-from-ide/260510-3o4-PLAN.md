<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> BFF proxy | Untrusted agent name in URL parameter |
| BFF proxy -> Ingest | Internal HTTP call, but still validate agent name |
| Ingest -> Filesystem | Agent name used to construct filesystem path (path traversal risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3o4-01 | Tampering | `agents/:name/avatar` endpoint | mitigate | Validate agent name against `^[a-zA-Z0-9_-]{1,64}$` regex before any filesystem operations |
| T-3o4-02 | Tampering | Avatar filename from IDENTITY.md | mitigate | Verify resolved avatar file path starts with workspace directory prefix to prevent path traversal via `../../` in Avatar field |
| T-3o4-03 | Information disclosure | Avatar endpoint error responses | accept | 404 responses are generic, no filesystem paths leaked |
| T-3o4-04 | Denial of service | Avatar image reads | accept | File reads are synchronous and small (avatar images), unlikely bottleneck |
</threat_model>
