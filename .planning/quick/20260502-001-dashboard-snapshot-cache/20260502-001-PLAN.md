# Quick Task 20260502-001: Dashboard Snapshot Cache

## Goal

刷新后秒开 Dashboard：先从 localStorage 恢复上次数据并显示 SYNCING 标记，后台 WS 连接后用新数据替换。

## Constraints

- `stores/gateway/gateway-store.ts` — 只允许**加** action/import，禁止改现有逻辑
- `gateway/` 目录不动
- SSR 安全：所有 localStorage 访问必须在 `typeof window !== "undefined"` 保护下

## Task 1: 创建 `lib/dashboard-snapshot-cache.ts`

**文件**: `lib/dashboard-snapshot-cache.ts`（新建）

**参考**: `references/openclaw-visual-agent-office/src/stores/gateway/agent-avatar-cache.ts` — 复用相同的 TTL + 版本 + 内存层 + SSR 安全模式。

**存储的字段**（序列化快照）：

| 字段 | 类型 | 备注 |
|------|------|------|
| `agents` | `[string, AgentInfo][]` | Map 序列化为 entries 数组 |
| `skills` | `SkillInfo[]` | |
| `cronTasks` | `CronTask[]` | |
| `channels` | `ChannelInfo[]` | |
| `providers` | `UsageProviderInfo[]` | |
| `usageDetail` | `UsageDetailSnapshot \| null` | |

**不存储**（实时数据，无意义缓存）：`agentLogs`、`globalEventFeed`、`alertItems`、`activeSessions`、`connectionStatus`

**实现骨架**：

```typescript
const SNAPSHOT_KEY = "ovao.dashboard.snapshot.v1"
const SNAPSHOT_VERSION = 1
```

## Task 2: 在 `gateway-store.ts` 中加 `hydrateFromCache` action

**文件**: `stores/gateway/gateway-store.ts`（修改——纯增量）

**修改内容**：

**① 在 `GatewayState` 接口加一行**（在 `ackAlert` 下面）：
```typescript
hydrateFromCache: () => void
```

**② 在 `fetchDashboardData` 末尾的 `set(...)` 调用之后，写入快照**：
```typescript
// fetchDashboardData 末尾 set({...}) 调用结束后追加：
writeDashboardSnapshot({
  agents: new Map(
    [...get().agents.entries()].filter(([, v]) => rpcAgents.has(v.id) || true)
  ),
  skills,
  cronTasks,
  channels,
  providers,
  usageDetail,
})
```

实际上更简洁的写法是在 `set(...)` callback 结束后直接用局部变量写：
```typescript
const snapshotAgents = new Map(rpcAgents)
writeDashboardSnapshot({ agents: snapshotAgents, skills, cronTasks, channels, providers, usageDetail })
```

**③ 在 store return 对象里加 action 实现**：
```typescript
hydrateFromCache: () => {
  const snapshot = readDashboardSnapshot()
  if (!snapshot) return
  set((state) => {
    // 只在 agents 为空时才 hydrate，避免覆盖已有的实时数据
    if (state.agents.size > 0) return {}
    return {
      agents: snapshot.agents,
      skills: snapshot.skills,
      cronTasks: snapshot.cronTasks,
      channels: snapshot.channels,
      providers: snapshot.providers,
      usageDetail: snapshot.usageDetail,
      isDashboardLoading: true,  // 标记为 loading — WS 连上后会覆盖
    }
  })
},
```

**验证**：TypeScript 无报错，store 类型检查通过。

## Task 3: 在 `GatewayBootstrap` 中先 hydrate 再 init

**文件**: `components/hud/gateway-bootstrap.tsx`（修改）

**修改内容**：

```typescript
'use client'

import { useEffect } from 'react'
import { useGatewayStore } from '@/stores/gateway/gateway-store'

export function GatewayBootstrap() {
  const init = useGatewayStore((state) => state.init)
  const disconnect = useGatewayStore((state) => state.disconnect)
  const hydrateFromCache = useGatewayStore((state) => state.hydrateFromCache)

  useEffect(() => {
    hydrateFromCache()  // 先恢复缓存（同步，立即生效）
    init()              // 再建立 WS 连接（异步）

    return () => {
      disconnect()
    }
  }, [disconnect, hydrateFromCache, init])

  return null
}
```

**验证**：组件 render，不报 hooks 错误。

## Task 4: 在状态栏加 SYNCING 指示器

**文件**: `components/hud/shell-status-bar.tsx`（修改）

**stale 状态判断**：`isDashboardLoading === true && agents.size > 0`
— 即"有缓存数据，但 WS 尚未刷新完毕"

**修改内容**：

```typescript
const isDashboardLoading = useGatewayStore((s) => s.isDashboardLoading)
const agentCount = useGatewayStore((s) => s.agents.size)
const isStale = isDashboardLoading && agentCount > 0
```

在状态栏左侧 WS 状态之后插入：
```tsx
{isStale && (
  <span className="text-[oklch(0.76_0.17_145)] animate-pulse">
    SYNCING
  </span>
)}
```

**视觉效果**：连接中且有旧数据时，状态栏显示 `SYNCING` 脉冲文字，连接完成后自动消失。

**验证**：刷新页面可看到 SYNCING 出现后消失；无缓存时不显示。
